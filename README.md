## üìñ About the Project

**CloudHabits** is a modern web application for tracking daily habits, specifically designed to work with Telegram Mini Apps. The application helps users build useful habits, track progress, and analyze completion statistics.

### üéØ Why CloudHabits?

- ‚úÖ **Ease of use** ‚Äî intuitive interface accessible directly from Telegram
- üìä **Progress visualization** ‚Äî calendar views and detailed statistics
- üî• **Motivation through streaks** ‚Äî tracking current and maximum completion streaks
- üì± **Mobile optimization** ‚Äî responsive design for comfortable use on any device
- üîí **Security** ‚Äî authentication via Telegram, isolated data for each user
- ‚ö° **Performance** ‚Äî fast operation thanks to optimized database queries

## ‚ú® Key Features

### üìÖ Calendar Views
- **Weekly view** ‚Äî quick overview of the current week with ability to mark habit completion
- **Monthly calendar** ‚Äî full month overview with day-by-day progress visualization

### üìà Reports and Statistics

- **Current streak** ‚Äî number of consecutive days without missing

- **Progress charts** ‚Äî visualization of dynamics for selected period (7 days, 30 days, 90 days)

### üé® Habit Management

- Integration with Telegram Mini Apps

## üîß API Endpoints

### GET `/`
Main page with weekly habit calendar.

**Parameters:**
- `user_id` (query) ‚Äî Telegram user ID

### GET `/calendar`
Monthly calendar with progress visualization.

**Parameters:**
- `user_id` (query) ‚Äî Telegram user ID
- `year` (query, optional) ‚Äî year
- `month` (query, optional) ‚Äî month

### GET `/reports`
Page with reports and statistics.

**Parameters:**
- `user_id` (query) ‚Äî Telegram user ID
- `period` (query, default: "7days") ‚Äî period: "7days", "30days", "90days"

### POST `/habits`
Create new habit.

**Form parameters:**
- `name` ‚Äî habit name
- `user_id` ‚Äî user ID

### DELETE `/habits/{habit_id}`
Delete habit.

### POST `/completions`
Toggle habit completion status for a specific date.

**Form parameters:**
- `habit_id` ‚Äî habit ID
- `date` ‚Äî date in YYYY-MM-DD format
- `user_id` ‚Äî user ID
- `context` ‚Äî context ("week" or "month")
